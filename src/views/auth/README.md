## ① 目的
- **本資料夾負責的角色 / 使用場景**：
  - 此資料夾 (`/auth`) 包含了所有與使用者身份驗證相關的前端介面元件。
  - `Auth.vue` 作為一個容器，管理「登入」與「註冊」兩個子元件的切換。
  - `Login.vue` 提供使用者登入的表單介面。
  - `Register.vue` 提供新使用者註冊的表單介面。

## ② 結構與命名規範
- **檔案/元件命名**：
  - 元件名稱使用大駝峰命名法 (PascalCase)，例如 `Login.vue`。
  - 檔案名稱與元件名稱保持一致。
- **輸出/匯入約定**：
  - 使用 `@` 作為 `src/` 目錄的別名。
- **不得事項**：
  - 不得在此層級的元件中，直接發起 API 請求。所有與後端或 Firebase 的通訊，必須透過 Pinia store (`/stores`) 進行。

## ③ 修改日誌 (Modification Log)

### `2024-07-29`
- **修改檔案**:
  - `Login.vue`
  - `Register.vue`
  - `Auth.vue`
  - `stores/authStore.js`
- **關聯**: 使用者註冊與登入流程優化
- **修改目的與原因**:
  1.  **密碼可見性切換**: 在 `Login.vue` 和 `Register.vue` 的所有密碼欄位中，新增了「眼睛」圖示，允許使用者點擊以顯示或隱藏密碼。此舉旨在降低使用者因輸入錯誤（如大小寫、符號）而導致的登入失敗或註冊挫折感，是提升基本可用性的關鍵功能。
  2.  **註冊成功後自動導航**: 改造了 `authStore.js`，使其在註冊成功後，能透過一個中央狀態 (`shouldSwitchToLogin`) 通知 `Auth.vue`。`Auth.vue` 監聽到此狀態變化後，會自動彈出成功提示，並將頁面從「註冊」切換至「登入」。這創造了一個無縫的引導流程，消除了使用者註冊成功後「不知下一步該做什麼」的困惑。
  3.  **登入選項 UI 改造**: 將 `Login.vue` 中原有的「Google 登入」按鈕，替換為一個視覺上更符合專案目標市場的「Line 快速登入」按鈕，並為其預留了功能開發接口。
  4.  **BUG 修復**: 修正了在 `Login.vue` 中，因手誤導致的 `</v--text-field>` 標籤錯誤，確保了應用的正常編譯與渲染。
